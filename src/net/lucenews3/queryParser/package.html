Implements a visitor pattern model for the Lucene query parser.

Suppose there exists an application which accepts user query strings and parses them using the Lucene query parser. Suppose the user searches for the string "Lord of the Rings". Internally, the application parses this string using "title" as the default field. Assuming a StandardAnalyzer is used, the Lucene query parser will produce a query object whose textual representation is now "title:lord title:rings".

The initial purpose of this package was to develop a query parser which produces queries that contain references to the original text that created them.

It was thought that a subclass of QueryParser would be suitable. Subclasses would be able to override the various get*Query methods (see QueryParser Javadoc) and use getToken(int) to determine the parsing token which created the current query.

Although this practice worked well initially, it was seen that the ability to turn the QueryParser into a pipe-and-filter mechanism would prove too difficult. This eliminated the possibility of having filtered QueryParser objects.

The QueryParser class invokes protected methods as it parses text. These protected methods are of the general form get[type]Query(...) where [type] represents the various categories of category (e.g. - fuzzy, term, field, boolean). All such methods return Query objects.
